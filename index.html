Current SUI Protocol 
I am using the Speyer UI System (SUI). Strictly follow these constraints:

1) Tokens only:
   - Use ONLY CSS variables defined in sui-tokens.css (e.g., --sui-bg, --sui-surface, --sui-border, --sui-blue, --sui-blue-hover, --sui-blue-soft, --sui-text, --sui-subtext, --sui-muted, --sui-success, --sui-warning, --sui-error, --sui-info).
   - Do NOT use arbitrary hex colors in component styling.

2) Typography (strict):
   - Font: Inter
   - Allowed weights: 400, 600, 700 only
   - H1: 32px / 700
   - H2: 24px / 600
   - H3: 20px / 600

3) Accessibility (non-negotiable):
   - No color alone. Every status MUST include an icon + text label.
   - Maintain strong contrast and readable spacing.
   - Use focus-visible outlines (3px) with --sui-blue-soft.

4) Geometry:
   - Buttons/inputs: 12px radius, 44px height
   - Cards: 16px radius
   - Use an 8px spacing grid

5) Output expectations:
   - Mobile-first layout, responsive breakpoints
   - Simple, predictable components
   - Avoid “clever” UI; prioritize clarity and consistency

How to Prompt

Paste the protocol, then describe your screen. Example: “Build a settings page in SUI with profile, security, and notifications.”

Maintaining Token Integrity

Tell the AI: “Always use SUI variables. Do not introduce new colors or Tailwind defaults.”
