Living Specification

A strict, token-driven UI system focused on clarity, accessibility, and consistency.

Philosophy

Speyer UI is designed to be easy to read, mobile-first, and predictable. Tokens are the contract. Components are simple, repeatable patterns.

No Color Alone
Status must include icon + label, not just color.
Token Fidelity
All colors come from --sui-* variables.
Geometry
Buttons/inputs: 12px radius. Cards: 16px radius. 8px grid.

For Designers

How to use SUI tokens in Figma (or any design tool) to keep handoff predictable and accessible.

Token-first workflow

Design in layers: background → surface/card → border → text → accents. This mirrors the token structure, so implementation stays consistent.

Contrast rule
Target 4.5:1 for body text. Use token pairs: --sui-text-primary on --sui-bg-card.
Red/green safe
Never rely on color alone. Always pair icon + label + (optional) shape.
Spacing system
Use the 8px grid tokens: --sui-space-2, --sui-space-3, --sui-space-4.

AI Prompt

Use this as a system instruction so AI-generated UI stays consistent with SUI. 

I am using the Speyer UI System (SUI). Strictly follow these constraints:

1) Tokens: Use ONLY CSS variables from sui-tokens.css (var(--sui-*)).
2) Theme: Dark mode is applied via [data-theme="dark"] on the html element.
3) Typography: Inter. Weights: 400/600/700. H1 32px, H2 24px, H3 20px.
4) No-Color-Alone: Every status MUST include icon + text label (and preferably shape).
5) Geometry: Inputs/buttons 12px radius; cards 16px radius; use 8px spacing grid.
6) Accessibility: Ensure readable contrast and clear focus rings. Avoid relying on red/green alone.

Build a UI that is clean, predictable, mobile-friendly, and strictly token-driven.
