For Designers

A practical handoff format: tokens, spacing, components, and accessibility rules — without ambiguity.

Design Rules

  • Mobile-first: layout collapses cleanly without “special cases”.
  • 8px grid: spacing is predictable across screens.
  • Token-only colors: no one-off hex values.
  • Readable typography: Inter 400/600/700 only.

Color Blind Safety

Designed for red/green color blindness: statuses never rely on color alone. Every state has an icon + label, and contrast is kept high for text and interactive elements.

Current SUI Protocol
I am using the Speyer UI System (SUI). Strictly follow these constraints:

1) Tokens are the contract. All colors MUST come from ./sui-tokens.css variables (no one-off hex values).
2) Typography: Inter weights 400/600/700 only. H1 32px, H2 24px, H3 20px.
3) Geometry: Buttons/inputs radius 12px, cards radius 16px, 8px spacing grid.
4) Accessibility: No status uses color alone — every state includes an icon + text label.
5) Dark mode: Use data-theme="dark" on the root element and rely on tokens for switching.

Build simple, readable UI. Avoid cleverness. Prioritize clarity and predictability.

Repo token file from this URL:
https://raw.githubusercontent.com/adrianspeyer/speyer-ui/main/sui-tokens.css

Use the CSS custom properties defined in that file.

Tip: include “red/green color blindness safe” as a hard requirement — then enforce icon + label everywhere.
