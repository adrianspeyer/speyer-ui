Core Principles

Every decision in SUI flows from these five principles.

Readability

Text and data are easy to scan and understand at a glance.

Accessibility

WCAG 2.1 AA compliant. Works for everyone, including screen readers.

Mobile-First

Built for small screens first. 44px touch targets. Scales up gracefully.

Color-Blind Friendly

Color is never the only indicator. Every status uses icon + text.

Consistency

Same components, same behavior, same look — everywhere.

AI Integration

No AI has SUI in its training data. Copy one of these prompts to teach any AI assistant how to build with the Speyer UI System.

Quick Prompt

Recommended for most tasks. Paste this before describing what you want built.

System Prompt
I want you to use the Speyer UI System (SUI) for this project.

## Step 1: Fetch the design tokens
Fetch the official SUI CSS token file from this URL:
https://raw.githubusercontent.com/adrianspeyer/speyer-ui/main/sui-tokens.css

Use the CSS custom properties defined in that file as the foundation for all styling. All tokens use the --sui- prefix.

## Step 2: Confirm display mode
Before writing any code, ask me:
"Would you like light mode only, dark mode only, or both (with a toggle)?"
Then implement accordingly:
- Light mode = default :root tokens
- Dark mode = [data-theme="dark"] tokens
- Both = include both token sets + a toggle using the data-theme attribute

## Step 3: Apply SUI design rules
Core Principles: readability, accessibility, mobile-friendly, color-blind friendly, consistency.

Typography: Inter font (Google Fonts). Fallback: system-ui.
Scale: H1 32px/700, H2 24px/600, H3 20px/600, Body 16px/400, Small 14px/400, Meta 12px/400.

Layout: 8px spacing grid (--sui-space-1 through --sui-space-6). Mobile-first. Max width: 1280px. Min touch target: 44px.

Components:
- Buttons: 44px min height, 12px border-radius, visible focus states
- Cards: 16px border-radius, border-based separation, 16-24px padding
- Inputs: visible borders, focus ring, minimum 44px height
- Alerts: ALWAYS include icon + text label (never color alone)

Accessibility (mandatory):
- WCAG 2.1 AA contrast ratios
- Color must NEVER be the only indicator of status
- Color-blind friendly: no red/green-only patterns
- Keyboard navigation, visible focus states, ARIA labels
- Respect prefers-reduced-motion

Status patterns (always icon + text):
- Success: ✔ + text | Warning: ⚠ + text | Error: ✕ + text | Info: ℹ + text

Motion: 150-250ms ease-out. Feedback only. No decorative animations.

Full docs: https://github.com/adrianspeyer/speyer-ui

System Prompt

For developers adding SUI to custom instructions, Cursor rules, or tool configs.

Custom Instructions
When the user says "use SUI", "use the Speyer UI System", or "Speyer UI":

1. Fetch the SUI design tokens from:
   https://raw.githubusercontent.com/adrianspeyer/speyer-ui/main/sui-tokens.css
2. Ask the user: "Would you like light mode, dark mode, or both?"
3. Apply all CSS custom properties (--sui- prefix) from the token file.
4. Follow these design rules:
   - Font: Inter (Google Fonts), fallback system-ui
   - 8px spacing grid (--sui-space-1 through --sui-space-6)
   - Min touch target: 44px, max content width: 1280px
   - Buttons: 44px height, 12px radius | Cards: 16px radius, border separation
   - WCAG 2.1 AA contrast | Keyboard nav | prefers-reduced-motion support
   - CRITICAL: Never use color alone for status. Always pair with icon + text.
   - Color-blind safe: no red/green-only patterns
5. Reference: https://github.com/adrianspeyer/speyer-ui

Tips for best results: Always paste the prompt first, before describing what you want built. If the AI cannot fetch the CSS file, paste the contents of sui-tokens.css directly into the conversation. SUI works with any framework — React, Vue, Tailwind, plain HTML. For follow-up prompts, just say "continue using SUI."
